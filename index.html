Kylin Overview

An open source distributed Analytical Engine from eBay to provide SQL interface and multi-dimensional analysis (OLAP) on Hadoop to support TB to PB size analysis.

Ecosystem

    Kylin Core: fundamental framework of Kylin OLAP Engine, including metadata engine, query engine, job engine, storage engine to run the entire stack, also with REST Server to run as service to serve client requests.

    Extension: plugin to add addtional functions or replace with specified feature to support different use cases.

    Integration: interactive with other application, system to support end to end solution, like scheduling system, ETL, monitoring and alert.

    Interface: User interface to enhance experience, bring more advanced features and plugin of other application.

    Driver: ODBC and JDBC driver to support different clients espcially Business Intelligence tool, for example Tableau.

What Kylin can do and can't do

Kylin is design to resolve query latency on Hadoop and offer an ASNI-SQL Interface to enable BI tool to interactive with data on Hadoop. Kylin should be interactive from Business Intelligence Tool, for example: Tableau, MicroStrategy and Excel, even 3rd party web application.


What Kylin can do:
  • ASNI-SQL on Hadoop: Kylin support most of ANSI-SQL query functions.
  • Interactive Query: User can get more better performance when query same Hive Query which covered by Kylin Cubes.
  • Pre-Build Data Model: User can define a data model and pre-build in Kylin with more than 10+ billions of raw data records.

    • What Kylin can't do:
  • ETL, Kylin will not cover ETL process.
  • Data Dump: Do not pull large data to client from Kylin in one single query. For example, one single query to get millions records to client will raise exception.
  • Data Mining: Kylin is not design for data mining relative work.
